# Master profile contract

`resume.md` is the master profile.

- Anything in the JD that cannot be traced to `resume.md` → **Do not add** section only.
- Tailored resume is a **view** of the master profile for one role, not a second truth source.
- Do not delete rare but true skills from base—only de-emphasize in tailored file if irrelevant.
- Inline notes in `resume.md` (e.g. "frame as...", "do not phrase as...",
  "not as tests she ran") are binding wording constraints, not background
  color — equal in priority to the hard boundaries in `anti-patterns.md`.
  Before using any fact or number attached to such a note, apply the framing
  exactly as written. If a note restricts how a fact can be phrased, no
  bullet or summary sentence may imply beyond what the note allows.
